**1. Plan Configuration:**
```typescript
export const PLANS = {
  starter: { name: 'BrewClaw Starter', amount: 19900, description: '1.5GB RAM, Priority Support' },
  pro: { name: 'BrewClaw Pro', amount: 49900, description: '3GB RAM, Priority Support' },

```

**2. Lazy-Initialized Client:**
Use lazy initialization to avoid build-time errors when env vars not present:
```typescript
let razorpayInstance: Razorpay | null = null

function getRazorpay(): Razorpay {
  if (!razorpayInstance) {
    const keyId = process.env.RAZORPAY_KEY_ID
    const keySecret = process.env.RAZORPAY_KEY_SECRET
    if (!keyId || !keySecret) {
      throw new Error('Razorpay credentials not configured')
    }
    razorpayInstance = new Razorpay({ key_id: keyId, key_secret: keySecret })
  }
  return razorpayInstance
}
```

**3. Customer Creation:**
```typescript
export async function createCustomer(email: string, name?: string | null): Promise<RazorpayCustomer>
```
- Creates or retrieves Razorpay customer by email
- Returns customer object with id

**4. Subscription Creation:**
```typescript
export async function createSubscription(
  customerId: string,
  planId: PlanId,
  options: { userId?: string; trialDays?: number }
): Promise<RazorpaySubscription>
```
- Uses `process.env[RAZORPAY_PLAN_${planId.toUpperCase()}]` for plan IDs
- Supports trial periods by setting `start_at` to future timestamp
- Includes userId in notes for webhook correlation

**5. CRITICAL - Timing-Safe Payment Signature Verification:**
```typescript
export function verifyPaymentSignature(
  subscriptionId: string,
  paymentId: string,
  signature: string
): boolean
```
- Signature format: `payment_id|subscription_id` (Razorpay subscription format)
- Use `crypto.createHmac('sha256', keySecret).update(...).digest('hex')`
- MUST use `crypto.timingSafeEqual()` for comparison (prevents timing attacks)
- Wrap in try-catch to handle length mismatch (return false)

**6. CRITICAL - Timing-Safe Webhook Signature Verification:**
```typescript
export function verifyWebhookSignature(body: string, signature: string): boolean
```
- Uses RAZORPAY_WEBHOOK_SECRET
- Same timing-safe comparison pattern as payment signature

**7. Status Mapping:**
```typescript
export function mapSubscriptionStatus(razorpayStatus: string): SubscriptionStatus
```
- Maps Razorpay statuses (created, authenticated, active, pending, halted, cancelled, completed, expired, paused) to app statuses (PENDING, TRIAL, ACTIVE, PAST_DUE, SUSPENDED, CANCELLED)

Follow the pattern from 2openclaw reference implementation exactly - this is battle-tested production code.

model Subscription {
  id                      String    @id @default(cuid())
  userId                  String    @unique

}
