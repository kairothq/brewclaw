  <files>app/api/subscriptions/create/route.ts</files>
  <action>
Create `app/api/subscriptions/create/route.ts` - POST endpoint for subscription creation.

**Request body:**
```typescript
{
  planId: 'starter' | 'pro' | 'business'
  trial?: boolean  // Optional 7-day trial
}
```

**Implementation:**

1. **Get authenticated user from NextAuth session:**
```typescript
import { auth } from '@/lib/auth'

const session = await auth()
if (!session?.user?.id || !session?.user?.email) {
  return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
}
```

2. **Validate planId:**
```typescript
import { PLANS, PlanId, createCustomer, createSubscription } from '@/lib/razorpay'

if (!planId || !Object.keys(PLANS).includes(planId)) {
  return NextResponse.json({ error: 'Invalid plan' }, { status: 400 })
}
```

3. **Create Razorpay customer:**
```typescript
const customer = await createCustomer(session.user.email, session.user.name)
```

4. **Create subscription with optional trial:**
```typescript
const trialDays = trial ? 7 : undefined
const subscription = await createSubscription(
  customer.id,
  planId as PlanId,
  { userId: session.user.id, trialDays }
)
```

5. **Save to Prisma database (status: PENDING or TRIAL):**
```typescript
await prisma.subscription.create({
  data: {
    userId: session.user.id,
    razorpayCustomerId: customer.id,
    razorpaySubscriptionId: subscription.id,
    plan: planId,
    status: trial ? 'TRIAL' : 'PENDING',
    trialEndsAt: trial ? new Date(Date.now() + 7 * 24 * 60 * 60 * 1000) : null
  }
})
```

6. **Return subscription details for frontend checkout:**
```typescript
return NextResponse.json({
  success: true,
  subscriptionId: subscription.id,
  shortUrl: subscription.short_url,  // Razorpay hosted checkout URL
  status: subscription.status,
  isTrial: !!trial
})
```

**Error handling:**
- Wrap in try-catch
- Log errors with sanitized data (no PII in logs)
- Return `{ success: false, error: message }` with status 500

<task type="auto">
  <name>Task 2: Create Payment Verification Route</name>
  <files>app/api/subscriptions/verify/route.ts</files>
  <action>
Create `app/api/subscriptions/verify/route.ts` - POST endpoint for payment verification.

**Request body (from Razorpay checkout callback):**
```typescript
{
  razorpay_payment_id: string
  razorpay_subscription_id: string
  razorpay_signature: string
}
```

**Implementation:**

1. **Get authenticated user:**
```typescript
const session = await auth()
if (!session?.user?.id) {
  return NextResponse.json({ error: 'Unauthorized', verified: false }, { status: 401 })
}
```

2. **Validate required fields:**
```typescript
const { razorpay_payment_id, razorpay_subscription_id, razorpay_signature } = body

if (!razorpay_payment_id || !razorpay_subscription_id || !razorpay_signature) {
  return NextResponse.json(
    { error: 'Missing payment verification fields', verified: false },
    { status: 400 }
  )
}
```

3. **CRITICAL - Verify signature with timing-safe comparison:**
```typescript
import { verifyPaymentSignature } from '@/lib/razorpay'

const isValid = verifyPaymentSignature(
  razorpay_subscription_id,
  razorpay_payment_id,
  razorpay_signature
)

if (!isValid) {
  console.error('[subscriptions] Invalid payment signature')
  return NextResponse.json(
    { error: 'Invalid payment signature', verified: false },
    { status: 400 }
  )
}
```

4. **Update subscription status to ACTIVE:**
```typescript
await prisma.subscription.update({
  where: { razorpaySubscriptionId: razorpay_subscription_id },
  data: {
    status: 'ACTIVE',
    razorpayPaymentId: razorpay_payment_id
  }
})
```

5. **Trigger container provisioning (PAY-03 - only after verification):**
```typescript
// Call provision endpoint internally
// In production, this calls GCP API
const provisionResponse = await fetch(
  `${process.env.NEXT_PUBLIC_BASE_URL || 'http://localhost:3000'}/api/provision`,
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      userId: session.user.id,
      subscriptionId: razorpay_subscription_id
    })
  }
)
```

6. **Return verification result:**
```typescript
return NextResponse.json({
  verified: true,
  paymentId: razorpay_payment_id,
  provisioned: provisionResponse.ok
})
```

**Security note:** The signature verification MUST happen before any provisioning. This is a security requirement (PAY-02, PAY-03) - never provision containers without verified payment.
