**Webhook events to handle:**
- `subscription.activated` - First successful charge, mark ACTIVE
- `subscription.charged` - Recurring charge succeeded
- `subscription.halted` - Payment failed, mark PAST_DUE
- `subscription.cancelled` - User or system cancelled, mark CANCELLED

**Implementation:**

1. **Get signature from header:**
```typescript
const signature = request.headers.get('x-razorpay-signature')
if (!signature) {
  console.error('[razorpay-webhook] Missing signature header')
  return NextResponse.json({ error: 'Missing signature' }, { status: 400 })
}
```

2. **CRITICAL - Get raw body BEFORE parsing (signature verification requires raw body):**
```typescript
const bodyText = await request.text()
```

3. **Verify webhook signature:**
```typescript
import { verifyWebhookSignature, mapSubscriptionStatus } from '@/lib/razorpay'

const isValid = verifyWebhookSignature(bodyText, signature)
if (!isValid) {
  console.error('[razorpay-webhook] Invalid signature')
  return NextResponse.json({ error: 'Invalid signature' }, { status: 400 })
}
```

4. **IDEMPOTENCY - Check if event already processed:**
```typescript
const eventId = request.headers.get('x-razorpay-event-id')
if (eventId) {
  const existingEvent = await prisma.webhookEvent.findUnique({
    where: { eventId }
  })
  if (existingEvent) {
    console.log(`[razorpay-webhook] Duplicate event ${eventId}, ignoring`)
    return NextResponse.json({ received: true, duplicate: true })
  }
}
```

5. **Parse event and handle by type:**
```typescript
const event = JSON.parse(bodyText)
const eventType = event.event
const payload = event.payload

console.log(`[razorpay-webhook] Received event: ${eventType}`)

switch (eventType) {
  case 'subscription.activated':
    await handleSubscriptionActivated(payload)
    break
  case 'subscription.charged':
    await handleSubscriptionCharged(payload)
    break
  case 'subscription.halted':
    await handleSubscriptionHalted(payload)
    break
  case 'subscription.cancelled':
    await handleSubscriptionCancelled(payload)
    break
  default:
    console.log(`[razorpay-webhook] Unhandled event: ${eventType}`)
}
```

6. **Save event for idempotency:**
```typescript

  await prisma.webhookEvent.create({
    data: { eventId, eventType, processedAt: new Date() }
  })
```

**Event handlers:**

```typescript
async function handleSubscriptionActivated(payload: any) {
  const subscription = payload.subscription?.entity
  if (!subscription) return

  await prisma.subscription.update({
    where: { razorpaySubscriptionId: subscription.id },
    data: {
      status: 'ACTIVE',
      currentPeriodEnd: subscription.current_end
        ? new Date(subscription.current_end * 1000)
        : null,
      nextBillingDate: subscription.charge_at
        ? new Date(subscription.charge_at * 1000)
        : null
    }
  })
  console.log(`[razorpay-webhook] Subscription activated: ${subscription.id}`)
}

async function handleSubscriptionCharged(payload: any) {
  const subscription = payload.subscription?.entity
  if (!subscription) return

  await prisma.subscription.update({
    where: { razorpaySubscriptionId: subscription.id },
    data: {
      status: 'ACTIVE',
      currentPeriodEnd: subscription.current_end
        ? new Date(subscription.current_end * 1000)
        : null,
      nextBillingDate: subscription.charge_at
        ? new Date(subscription.charge_at * 1000)
        : null
    }
  })
  console.log(`[razorpay-webhook] Subscription charged: ${subscription.id}`)
}

async function handleSubscriptionHalted(payload: any) {
  const subscription = payload.subscription?.entity
  if (!subscription) return

  await prisma.subscription.update({
    where: { razorpaySubscriptionId: subscription.id },
    data: { status: 'PAST_DUE' }
  })
  console.log(`[razorpay-webhook] Subscription halted: ${subscription.id}`)
}

async function handleSubscriptionCancelled(payload: any) {
  const subscription = payload.subscription?.entity
  if (!subscription) return

  await prisma.subscription.update({
    where: { razorpaySubscriptionId: subscription.id },
    data: { status: 'CANCELLED' }
  })
  console.log(`[razorpay-webhook] Subscription cancelled: ${subscription.id}`)
}
```

  <name>Task 2: Update Middleware to Exclude Webhooks</name>
  <files>middleware.ts</files>
  <action>
Update the existing `middleware.ts` to exclude webhook routes from authentication.

**Current matcher:**
```typescript
matcher: [
  '/((?!api/auth|_next/static|_next/image|favicon.ico|public).*)',
]
```

**Updated matcher (add api/webhooks):**
```typescript
matcher: [
  '/((?!api/auth|api/webhooks|_next/static|_next/image|favicon.ico|public).*)',
]
```

The key change is adding `api/webhooks` to the exclusion list. Without this, Razorpay webhook calls would get 401 Unauthorized because they can't authenticate as NextAuth users.

**Why this is critical (from research):**
- Razorpay webhooks are server-to-server calls
- They include signature headers but no NextAuth session
- Middleware intercepts before route handler
- If 401 returned, Razorpay marks delivery as failed and retries
- Webhook signature verification provides authentication instead
